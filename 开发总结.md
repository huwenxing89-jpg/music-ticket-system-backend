## 核心技术实现

### 座位锁定机制
- 使用Redis实现分布式锁
- 座位状态：0-可选，1-已锁定，2-已售出
- 锁定时间：15分钟
- 定时任务清理过期锁

### 订单状态流转
```
待支付 --支付--> 已支付 --完成--> 已完成
   |                         |
 超时取消                   申请退款
   |                         |
已取消 <---------       退款中 --> 已退款
```

### JWT认证流程
1. 用户登录 -> 生成Token
2. 请求接口 -> 携带Token -> 验证Token
3. Token过期 -> 刷新Token
